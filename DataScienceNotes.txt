What is P Value ->

How do we define Confidence Interval? There is a very important parameter called as Significance Value. Significance Value is denoted by Alpha. If we say Alpha is 0.05, then we have to find the Confidence Interval by taking 1-Alpha(1-0.05), which is 0.95.

Z Test - Hypothesis Testing ->

Whenever we want to use the Z Test, then we need to know the Population Standard Deviation, and the sample size should be greater than or equal to 30.
Z Table gives the value of the Area under the curve.
We find the Z Score value in the Z Table by using the Area under the curve.
If the value comes between the range of negative Z Score value and positive Z Score value, we will consider it as the Null Hypothesis (H0) is true and we fail to reject the null Hypothesis. If it comes outside the Z Score value, then we reject the null Hypothesis.
When we get the new Z Score value, we construct a new graph by using the new value, and we find the area under the curve by checking the Z Table.
We can calculate the P Value by adding the vaule of area under the curve on both the sides, and if it is less than the Significance Value, then we reject the Null Hypothesis. If the value is greater than the Significance Value, then we accept the null hypothesis or we fail to reject the null hypothesis.
Z Test usually gets applied for the average data.

Analysis of Variance (ANOVA Test) ->

ANOVA is a statistical method used to compare the means of 2 or more groups.
Wrt ANOVA we have 2 components, the first component is Factors, the Factors can be dependent or independent. The second component is Levels.
Factor could be Mode of Payment and Levels could be GPay, PhonePe, IMPS or NEFT.

Assumptions in ANOVA ->

1. Normality of Sampling Distribution of Mean. The distribution of sample mean is normally distributed.
2. Absence of Outliers. Outliers needs to be removed from the dataset if present.
3. Homogenity of Variance. Population Variance in different levels of each independent variable should be same.
4. Samples are independent and randomly selected.

Types of ANOVA ->

There are 3 types of ANOVA:
1. One way ANOVA. One Factor with atleast 2 Levels, these Levels are independent.
2. Repeated Measures ANOVA. One Factor with atleast 2 Levels, theses Levels are dependent.
3. Factorial ANOVA. Two or more Factors (each of which with atleast) 2 Levels. Levels can be independent or dependent.

Partitioning of Variance in ANOVA ->

To calculate ANOVA we use the F Table, F = Variance between Samples / Variance within Samples

Understanding Simple Linear Equations ->

With the unit movement in the x-axis, what is the movement with respect to the y-axis, that indicates something called as slope.
Error is nothing but the difference between the true points and the predicted points.
We try to find out the optimized way, where in we create a best fit line and then we try rotate this best fit line by changing the values of intercept and slope and then try to find out the best fit line with minimal error.

Cost Function ->

Cost Function is used find the best fit line.
Our main aim is to Minimize the Cost Function.
We can minimize the Cost Function by changing the values of titha0 and titha1, and we try to find out the best fit line where in we do the summation of the particular error and this should be minimal.
We bassically say this Cost Function as Mean Squared Error.
Equation of the straight line is htitha(x) = titha0 + titha1x
If titha0 = 0, that basically means, when x value is 0, it is basically passing through the origin, hence the intercept is exactly 0.

The dataset has x=(1,2,3) and y=(1,2,3).
When titha0 = 0, the equation becomes htitha(x) = titha1x
When titha1 = 1, the Cost Function Jtitha1 is 0 and there is no error, the best fit line passes through all the points.
When titha1 = 0.5, the Cost Function Jtitha1 is 0.58.
When titha1 = 0, the Cost Function Jtitha1 is 2.3.

If we continue to do it with different titha1 value, we will get a curve in the graph. The point where the error was very less is called the Global Minima, and we really need to achieve Global Minima or atleast near to that point to say that my error has minimized. The curve is called as Gradient Descent. The Gradient Descent is important in Deep Learning techniques as well. We have to come to Global Minima point by changing titha0 or titha1 value.
We cannot be randomly selecting titha1 value, so we need to apply convergence algorithm.

Convergence Algorithm ->

Convergence Algorithm will actually help you to initialize one titha1 value, and automatically based on the Gradient Descent it should be able to increase or decrease the titha1 value. Convergence Algorithm basically says to repeat until the convergence means until we reach the Global Minima.

The equation is tithaj = tithaj - alpha(derivative/derivative(tithaj))*J(titha1).

We will initalize some value to tithaj and we will try to find out the Cost Function.
The above equation will help us to change our titha1 value much more efficiently.

When we say derivative, we mean we are going to calculate the slope.
In order to find out the derivative, we need to find out whether it is a positive slope or negative slope because derivative is nothing but we are trying to find out the slope. So at a particular point in order to calculate the slope we create a tangent line. See the right side of the tangent line to find out whether it is a positive slope or negative slope. If we are able to find out whether it is a positive slope or negative slope, then we will be able to understand whether we have to reduce tithaj value or increase tithaj value.
If the right part of the line is facing downwards, we will say it as negative slope.
If we get a negative slope, then tithaj = tithaj - alpha(-ve value), (derivative/derivative(tithaj))*J(titha1) will be negative. It will become tithaj = titha + (+ve value) and it will increase the tithaj value.

If we do the above iterative process again and again, we will reach the Global Minima by optimizing the tithaj value.
If the right part of the tangent is pointing upward, then it becomes a positive slope. In case of positive slope we need to decrease the tithaj value, then only we will be able to reach the global minima, then tithaj = tithaj - alpha(+ve value), (derivative/derivative(tithaj))*J(titha1) will be positive. It will become tithaj = titha - (+ve value) and it will decrease the tihaj value.

Alpha is nothing but it is a learning rate. Learning rate controls the speed at which convergence happens. If it is a very small value then it will take more time to converge, if it is a higher value then it may never converge, so we have to take the small value but should not be very smaller.

Performance Metrics ->

Performance Metrics is used to determine whether the model is good or not for a specific problem statement.
Rsquared equation is:
Rsquared = 1 - Sum of Square Residual/Sum of Square Total, which can also be writte as:
Rsquared = 1 - Summation(yi - yihat)^2/Summation(yi - yibar)^2,
where,
yi is the true point.
yibar is the predicted point.
yihat is the average of true point.

The numerator difference will be smaller number when compared to denominator difference.
Rsquared equation gives us the percentage of accuracy and shows us how our model is performing. More the value towards 1, more accurate the Linear Regression model is.

When we have good positive correlation, then the Rsquared value increases.

The equation of the Rsquared is such that, as more the features increases the Rsquared value keeps increasing. If there is positive correlation of the independent feature with the dependent feature then the Rsquared value increases more. If there is no correlation of the independent feature with the dependent feature then the Rsquared value increases less. This is the problem with respect to Rsquared, eventhough there is no correlation the Rsquared value increases.
In order to prevent this, we use Adjusted Rsquared. Adjusted Rsquared penalizes with respect to every feature that are not correlated with the output feature.
The equation of Adjusted Rsquared is:
Adjusted Rsquared = 1 - (1-R^2)(N-1)/N-P-1, where N is number of data points and P is number of independent features.

MSE, MAE, RMSE ->

MSE: Mean Squared Error, MAE: Mean Absolute Error, RMSE: Root Mean Squared Error.

MSE equation:
MSE = Summation((y - yihat)^2/n), where n is the number of data points, y is the actual value of the observation and yihat is the predicted value of the ith observation.
Our best fit line must be created such that the Cost Function should keep on reducing.

(y - yihat)^2 is a quadratic equation because it is in the format of ax+by+c, (a - b)^2 = a^2 - 2ab + b^2.
Whenever we have a quadratic equation and if we plot the quadratic equation, then we will be getting Gradient Descent. In Gradient Descent we will be having Global Minima. Our aim is to come to the Global Minima or come closer to the Global Minima, because there the Error will be less.
The Gradient Descent is differentiable at all points. It is differentiable because it forms a Gradient Descent, so at every point we will be able to calculate the slope and based on that we can reduce the coefficient and reach the Global Minima.

If we have a local minima, the convergence will go ahead and stop over there because there the slope will be zero. But in real we need to reach the global minima.

In convex function we will be having one local minima or one global minima.
In non convex function we will be having both local minima and global minima.

Because of the outlier there would be change in the best fit line, that is why MSE is not robust to the outliers.

MAE equation:
MAE = 1/nSummation(y - yihat)

RMSE equation:
RMSE = sqrt(1/nSummation(y - yihat)^2)

As a starter, we should try to check all of this.

Notes:
Mean Squared Error measures the amount of error in statistical models. It assesses the average squared difference between the observed and predicted values. When a model has no error, the MSE equals zero. As model error increases, its value increases.
Mean Absolute Error is a measure of the average size of the mistakes in a collection of predictions, without taking their direction into account. It is measured as the average absolute difference between the predicted values and the actual values and is used to assess the effectiveness of a regression model.
The root mean squared error measures the average difference between a statistical model's predicted values and the actual values. Mathematically, it is the standard deviation of the residuals.

Overfitting and Underfitting ->

While training the model with the training dataset, would never know about test dataset.
We further split the Training data to Train and Validation. Train part is used to train the model and Validation is used for Hyperparameter Tuning the Model. We combine Train and Validation data in order to make the model performance better.

When the model gives very good accuracy to the Training Data and very good accuracy to the Test Data, such model is known as Generalized model as it is working well with both Training and Test Data.

Very Good Accuracy is Low Bias in Training Data and Very Good Accuracy is Low Variance in Test Data.

Suppose we have a scenario where the model gives very good accuracy(90%) to the Training Data but bad accuracy(50%) to the Test Data, then we say that the Model is Overfitting.
We say that the Model is Overfitting when the Model has been trained well with the Training Dataset but with respect to the new data points it is not able to predict.
Very Good accuracy is Low Biased in Training Data and Bad Accuracy is High Variance in Test Data.

When the model gives Low Accuracy with respect to the Training Data and Low Accuracy with respect to the Test Data, then we say that the Model is Underfitting.
Low Accuracy is High Bias in Training Data and Low Accuracy is High Variance in Test Data.

We should always try to find out the Generalized Model where the Training(90%) and Test Accuracy(85%) should be good.

Ridge Regression, Lasso Regression and ElasticNet Regression ML Algorithms =>

Ridge Regression ->

Ridge Regression is used in reducing the Overfitting.
If we get an accuracy of 100%, consider it as the model is overfitting. We should never get Training accuracy as 100%.
Ridge Regression is also known as L2 Regularization.

We can consider Ridge Regression as a new Algorithm which will help us to Hyperparameter Tune the Linear Regression.
Cost Function equation:
1/2mSummation(htitha(x) - y)^2 + lambdaSummation(Slope)^2.
When we are adding Lambda and Slope^2, the situation is going to be such that we are never going to get best fit line that passes exactly through the training data points. This is how Ridge Regression is making sure that the Overfitting does not happen.

When lambda is 0, we will have the the Cost Function of Linear Regression.
When we increase the value of lambda with respect to titha, it will get multiplied to slope^2, we will get new Gradient Descent and then you will be able to see that the Global Minima has changed. Lambda can be considered as Hyperparameter.
The relationship between lambda and slope is when lambda is increasing then the slope will decrease.

Ridge Regression is reducing Overfitting by reducing the impact by reducing the coefficient of the feature that are not directly corelated with the output feature. Here we can see x3 is hardly corelated by 0.24, and further it is trying to reduce the value and it will have less impact on the best fit line.

Lasso and ElasticNet ->

Lasoo Regression is also called as L1 Regularization. We use Lasso Regression for Feature Selection.
Cost Function equation:
1/2mSummation(htitha(x) - y)^2 + lambdaSummation|Slope|.
With the help of lambdaSummation|Slope| equation, we will try to do Feature Selection. Features that are not important will automatically get deleted and features that are important will be considered.

When the lambda value increases, the titha value keeps decreasing. But after 1 point of time, the titha value will become zero, therefore the coefficient is actually becoming zero. When the coefficient becomes zero, we are removing the feature.
We use Lasso Regression when we have many features, because automatically it is going to find out features that are not highly correlated, it will just remove it by making the coefficient to zero.

ElasticNet is the combination of both Ridge Regression and Lasso Regression.
We try to reduce overfitting and also do Feature Selection.
Cost Function equation:
1/2mSummation(htitha(x) - y)^2 + lambdaSummation(Slope)^2 + lambdaSummation|Slope|.
When we have a model that is Overfitting and has a lot of Features, then we can use ElasticNet Regression.
Hence Ridge, Lasso and ElasticNet are Hyperparameter Tuning the Linear Regression.

Introduction to Support Vector Machine ->

Support Vector Machine Algorithm helps us to solve both Classification and Regression problems. If we are solving a Classification problem, we basically say this as Support Vector Classifier(SVC) and in the case of Regression, we basically say Support Vector Regressor(SVR).
To understand Support Vector Machine, it is really important that we need to understand Logistic Regression.
In Logistic Regression, suppose if we have 2 categories of points, lets say it is a Binary Classification. In Logistic Regression our main aim is to create a best fit line, that is a line between the 2 particular points such that this line is able to clearly separate them, and this is what we basically do with respect to Logistic Regression.

In Support Vector Machine, in Support Vector Classifier, we create a best fit line in 2-dimension, in case of 3-dimension it becomes a plane, in case of n-dimension it becomes a hyperplane. In Support Vector Classifier, along with the best fit line, we also try to create 2 more lines and these lines are basically called as marginanl planes.
The marginal planes should be equal distance, and marginal planes should be created in such a way that the distance between this marginal plane should be maximum.

We need to select the best fit line with distance between the marginal plane should be maximum for doing the classification.

The another thing that we need to understand here is that, with the marginal plane passes through one of the nearest point from this particular plane, these points are basically called as Support Vectors.

The aim of Support Vector Machine is that, we need to create a best fit line along with we need to create the marginal planes, so that we will be able to categorize the point very clearly or classify the point very clearly.
In 3-dimensional plane also we need to create the plane along with the marginal planes and the distance between this marginal plane should be maximum.

Soft Margin and Hard Margin ->

In real world scenario, we will not have fully completely separable marinal points. There will be lot of overlapping. In overlapping we cannot clearly separate just by using best fit line and marginal plane, there would definitely be some amount of errors. Errors means the points may be coming in other category as well. This situation is basically caleed as Soft Margin. In Soft Margin, some amount errors will always be there.

In a scenario, where we are clearly separating all the points and we dont have any errors. This scenario is basically called as Hard Margin.

In Hard Margin we dont get errors, in soft margin we get some amount of errors because there is a lot of overlapping.

Naive Bayes Theorem =>

Understanding Baye's Theorem ->

Naive Bayes Algorithm is specifically used to solve Classification problems, both Binary and Multi Class Classification. In order to understand Naive Bayes, we need to have basic idea of Probability. Conditional Probability will be used in Naive Bayes Machine Learning Algorithm.

If in a generic way, we can write the Probability equation of Dependet or Conditional event:
P(A and B) = P(A) * P(B/A)

This is what we are going to use for deriving the Baye's Theorem.

Based on Baye's Theorem the entire Naive Baye's Algorithm works.

The below equation is called as Baye's Theorem:
P(A/B) = (P(A) * P(B/A)) / P(B)

We will understand how the above equation will be used in Machine Learning problem statement.

The same equation can be written for independen and dependent events like:
P(y/x1,x2,x3) = (P(y) * P(x1,x2,x3/y)) / P(x1,x2,x3)

The above equation is all about Naive Baye's Machine Learning Algorithm.

Refer the Course Material 1.0-Naive+Bayes+Classifier+Indepth+Intuition.pdf.

Naive Baye's Machine Learning Algorithm uses Baye's Theorem to probably calculate everything.
In short the Naive Baye's Algorithm calculates the probabilities like 'Yes' or 'No', if it is a Multi Class Classification based on the input features that we are getting.

The output is just a probability, but we should try to make this in the form of 100%.

Naive Baye's solves any kind of Classification Problem, let it be with respect to Binary Classification or Multi Classification.

Optimization of KNN - KD Tree and Ball Tree Indepth Intuition ->

KD Tree:
Our aim is that to find out the way that, whenever we try to find out the K Nearest Neighbour we should not go and check distance of each and every point.
For this, we will try to create a new tree which is called as KD Tree which is in the form of a Binary Tree Format. With the hlep of Binary Tree Format, we are reducing the number of searches.

Refer the Course Material 2.0-KD+Tree+Ball+Tree+.pdf.

Find the median of the Features.
Take the nearest point of the median value.

Once we get the Median value, we will draw a line projecting it into the x axis at the nearest point of the Median. Because of the line, we have 2 main regions. The region has come by finding out the Median value of the Feature.
We create a new line for the next Feature and projecting it into the y axis at the nearest point of the Median. We can continue further and find out the Median of the points present in the different regions and create a new line either projecting it into the x axis or into the y axis at the neares point of the Median.
By creating this we are actually creating a split, and when we are creating a split, the split will be cconsidered into the Binary Tree.

If any new point comes, and then we have to calculate the distance, it will come in a specific region and we should check that specific region. From that specific region, we have to find the nearest point by tracking the Binary Tree and back tracking the binary tree to find the next nearest point. By this, we decrease the Time Complexity.

Ball Tree:
In Ball Tree, we dont have to do the back tracking. In Ball Tree, the concept is very simple. Whichever is the nearest point, we group them together and create Groups.
Next we can combine the nearest Groups, and finally we combine the entire Group. We have combined them with respect to Clusters.
When we get a new data point, we will know which is the nearest neighbour point.
So this way because of the Group we are directly able to get the elements instead of searching each and every element and calculating the distance. By this, we decrease the Time Complecity.

Decision Tree Classifier and Regressor =>

Introduction to Decision Tree ->

Random Forest, Xg Boost uses Decision Tree internally.
Decision Tree can be used to solve both the kinds of Problem Statement, that is Classification and Regression Problem Statement.
Two types of Decision Tree Classifier:
1. ID3
2. Cart
The basic difference between ID3 and Cart is that, in Cart whenever we are creating a Decision Tree, we create only Binary Splits, whereas in ID3 we can create more than Binary Splits.
Decision Tree will be working like if, else if condition
Decision Tree will try to create in the bottom to top approach.

Impure Split is such a split which can further have Binary Split and does not have a leaf node.
Pure Split is such a split which has no further Binary Split and have a leaf node.
The Split continues to happen until and unless we get Leaf node.

For finding out mathematically whether the split is a pure split or an impure split, we can use Entropy and Gini Index.
If we select a particular feature and if we are able to get the leaf node quickly, that means the feature selected is correct.
But if we select a feature, that may not give a leaf node or it may take time to give you a leaf node, then that is bad.
So for this particular case, we use something called as Information Gain.

Entropy and Gini Impurity ->

p+ is nothing but probability of being positive, or 1.
p- is nothing but probability of being negative, or 0.

If we have a very impure split, when number of yes and no are equal. Then the maximum split value will have 1.
For the pure split, the maximum split value will have 0.
For the probability of + and -, the value will be between 0 and 1. If we say probability of + is 0.5, then probability of - is 0.5.

Whenever we have probability of + as 0.5 or probability of - is 0.5, the maximum valaue that we can get is 1. This is a complete impure split.
In case of Entropy, the values will always be between 0 and 1.

In case of Gini Impurity, if it is a complete impure split, the maximum value will be 0.5.
For the pure split, the maximum value will be 0.
In case of Gini Impurity, the values will always be between 0 and 0.5.

How do we decide which features to basically select to make the decision tree split? For this, we will try to understand about Information Gain.
With the help of Gini Index and Entropy, we are definitely able to find out whether the node split is pure split or impure split.

Information Gain ->

How do we determine which feature to select in order to start the split? For that we will be using Information Gain.

For Formula, refer the Course Material Decision+Tree+Classifier+.pdf.

When we have an impure split, then the Entropy of the Category will be 1.
We can select the feature which has the highest Gain to start the split.
Decision Tree internally checks all this conditions.
This is how Information Gain is calculated.

When should we use Entropy and when should we use Gini Index?

Entropy vs Gini Impurity ->

Whenever the dataset is small we can use Entropy. For example 10000 records.
With respect to small datasets, the difference between the time that is going to take to train the model will be minimal between Entropy and Gini Impurity.
When the dataset is large, then try to use Gini Impurity.

By default, whenever we use Decision Tree Classifier, then use Gini Impurity.

Decision Tree Split for Numerical Features ->

What if we get a feature, which has continuous values rather than categorical values. In this scenario how should we go ahead and do the decision split?
First we will sort the feature values in ascending order.
We create many Decision Trees, in every Decision Tree, we will have a Threshold, lets consider the Threshold will have the first value. The root node will have the first value.

To understand which split is better, we basically use the Information Gain. So whichever has the highest Information Gain we will be selecting that specific split.
The Time Complexity will usually be high when we will have millions of records.

Post Pruning and Pre Pruning ->

In Decision Tree, whenever we try to do the split completely to its depth with respect to the training dataset, usually we face a scenario called as Overfitting.
Overfitting basically means your train accuracy is very high because we have done splitting completely till the end. But with respect to the Test Accuracy, the accuracy will be very very low. This scenario we basically say it as Overfitting.
With Overfitting, we get Low Bias and High Variance. Bias is for Training data and Variance is for Test data.
In order to reduce Overfitting we can use 2 different techniques:
1. Post Pruning
2. Pre Pruning

Post Pruning basically means, we will construct the decision tree completely and then after that we can start Pruning it or Cutting it with respect to depth. We Prune it so that it does not lead to Overfitting scenario. Post Pruning should be applied for smaller datasets because the Time Complexity will be high when we have millions of records.

Pre Pruning basically means, while we are constructing the decision tree, we dont have to construct the decision tree entirely like how we did it in Post Pruning, hence while we are constructing the decision tree, we can play with some of the parameters. We have some parameters like max_features, max_depth. All these parameters are basically some of the Hyperparameters.
In short in Pre Pruning, we tune with the Hyperparameters while constructing decision tree.

Decision Tree Regression ->

Whenever the output feature is continuos feature at that point of time we use Decision Tree Regressor.
Entropy and Gini Impurity is basically for the purity split. Information Gain is basically to find out which feature should we use for making the split.

In Decision Tree Regressor, we cannot use Entropy, Gini Impurity and Information Gain.
Entropy, Gini Impurity and Information Gain, can be used when the output feature is categorical.

We create many Decision Trees, in every Decision Tree, we will have a Threshold, lets consider the Threshold will have the first value. The root node will have the first value.
In Decision Tree Classifier, we used to use Information Gain for deciding this. In Decision Tree Regressor, we cannot use this.
We are going to use something called as Variance Reduction to decide which split to use.
We are basically going to compute mean squared error for each and every node and then we are going to combine them to actually find out the variance reduction. Then finally out of all the splits, whichever has the maximum variance reduction, we are going to take that split.
When we are using Decision Tree Regressor, we use Mean Squared Error instead of using Entropy and Gini Impurity.

Variance Formula is = 1/nSummation(y - ybar)^2.
Variance Reduction Formula is = Var(Root) - Summation(W(i) * Var(Child)).
For Calculation, refer the Course Material Decision+Tree+Regressor.pdf.

Lets say, if we get new Test Data.
If it is less than the root node value, then it will go to the left node, how are we gonna find the final output?
Lets consider the left node to be the leaf node. The ouput will be the average of all the values present in that left node.
If it is greater than the root node value, then it will go to the right node.
The output will be the average of all the values present in the right node.

If we get Gini around 0.5, then basically it means an Impure Split. If we get Gini near to 0, then that becomes a very good Pure Split.
This kind of split will basically han from Top to Bottom.

In most of the platforms Kaggle and even in different Hackathons, many people while they are solving the Machine Learning Problem Statement, they use this kind of Ensembled Techniques, especially Bagging and Bossting Techniques. Some of the Algorithms that we will learn in Bagging is Random Forest.

Idea Behind Bagging:
In Bagging, suppose we have a specific Training Dataset. In Bagging Technique, we have multiple Base Learners. When we say Base Learners, this are basically different different Machine Learning Algorithms or it can be same Machine Learning Algorithms.
In Bagging, we give some sample of Dataset to the Model 1, and another sample of Dataset to Model 2, another sample of Dataset to Model 3, and another sample of Dataset to Model n.
Different Models will get trained from different Datasets. Once the Model is trained from different different sample of Datasets, then when we give out new Test Data for Predictions. When we pass this specific set of Data, the new test data will go to the Model 1, and then the Algorithm will do some kind of Predictions. When we want to find out the Final output in this specific way, then we use something called as Majority Voting Classifier in the case of Regression Problem Statement. This basically means, they will go and see that each and every Model which we specifically say as Base Learners, what output they are giving, like maximum number of Models are giving the output as 0. So our final output based on Majority Voting Classifier will be 0. This is the idea behind Bagging.

If we have a Regression Problem Statement, the ouput will be a continous value. So the final output will be the average of all the outputs.

Important thing about Bagging:
All the Base Learners are getting trained parallely, not sequentially.

Bagging and Boosting Algorithms are used for solving both Classification and Regression Problems.

Idea Behind Boosting:
In Boosting, we will learn about the different Algorithms:
1. AdaBoost Algorithm
2. Gradient Boost Algorithm
3. Extreme Gradient Boost Algorithm, also called as XgBoost

Fundamental idea behind Boosting:
In Baggin we had Base Learners, in Boostingwe basically say that we have Weak Learners.
We have the Dataset, and many Models which are connected Sequentially. We combine all the Weak Learner and finally when we combine all Sequentially, we actually get something called as Strong Learner.
In Boosting, we give the entire Dataset to Model M1. Model M1 will train and it will give some kind of Accuracy, lets say for some of the records it has not predicted correctly. It will pass those records which are incorrectly predicted along with some more Dataset to Model M2. Similarly Model M2 will train from that specific records and will train, and will produce some wrong records. It will pass all the wrong records along with some Dataset to Model M3. This process will keep on continuing with n number of Models.
When we try to give new Test Data, then each and every Model will have its own strength and based on that it will give the predictions. Overall when we combine all the predictions, again we can include Majority Voting Classifier or we can also get with respect to, if we try to find out the avearge we will able to get the Regression Problem Statement. In Short, in Boosting we are combining multiple Weak Leqarners together to form a Strong Learner.

Random Forest Regression ->

Lets consider a Dataset, and m features.
Random Forest uses the approach of Bagging, the base learners that we actually have, all the Models of the Base Learner are Decision Tree.
We are going to do some Row Sampling from the Dataset. From the Dataset, we are going to pick up some rows. Along with Row Sampling we are going to do something called as Feature Sampling.
Row Sampling means we are going to pick some amount of rows from the Dataset. Feature Sampling means we are going to pick some Features and give it Decision Tree Model 1 and it will get trained over there.
For the Model 2, we will be doing Row Sampling with replacement. We are doing replacement because, some of the rows may get repeated from Model 1 but we are also replacing some rows. So all the rows should be same as all the rows that we have given to Model 1. We may have to do Feature Sampling with replacement.
Some of the rows and features may get repeated, but majority of the rows and features will get replaced.
Same thing we are going to do for the other Models.

In Random Forest, we use only Decision Trees. All the Decision Trees will get trained with respect to all the training Dataset.

Whenever we try to give new Test Data, we are going to take the Majority Voting Classifier in the Classification.
With respect to Regression, the output will be Continuous, hence we calculate the average of all the Model outputs.

Why should we use Random Forest instead of Decision Tree?

In Decision Tree, we face a scenario called as Overfitting, with Overfitting we get Low Bias and High Variance.
With the help of Random Forest, since we have many Base Learners, the High Variance automatically will become Low Variance because our Test Accuracy will increase because we have many Base Learners that are trained on separate separate sample of Data bu using Row Sampling and Feature Sampling concepts.

What is Column Transformer?
Whenever we want to handle categorical features or we want to do Standard Scaling, we are specifically doing that in Columns.
Column Transformer is a kind of Library or Class, what it does os that, any kind of Transformation we give inside it, we can combine multiple Transformation technique.
For ex: We can combine OneHotEncoder and StandardScaler.
StandardScaler we will apply in Numerical Features.
OneHotEncoder we will apply in Categorical Features.
When we use OneHotEncoder with important property drop='first', if we have Binary Categories it will do One Hot Encoding for Binary, we will get 2 Columns. The number of Columns created by One Hot Encoding, 1 Column is not at all required.
Suppose there are n Columns, n-1 Columns will try to represent the entire data. So that is the reason, we will be dropping the First Column after we probably create One Hot Encoding.
So for this specific case, we are going to use Column Transformer.
Since we are combining multiple Transformer technique, we have to use Column Transformer.
Whenever we want to apply Transformation in Training Dataset, we have to basically apply fit_transform.
For the Test Dataset, whenever we need to apply the Transformation, we just use transform, because of data leakage.
